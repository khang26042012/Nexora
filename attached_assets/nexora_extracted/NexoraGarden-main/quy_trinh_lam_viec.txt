QUAN TRỌNG: Render tự động deploy mỗi khi có commit mới.
Nếu push nhiều commit liên tiếp → Render sẽ cancel hết, chỉ
giữ cái cuối cùng → gây lãng phí và log lộn xộn.

CÁCH PUSH ĐÚNG (dùng GitHub Git Tree API):
  1. Tạo blob cho từng file thay đổi
  2. Tạo 1 tree mới chứa tất cả file
  3. Tạo 1 commit duy nhất từ tree đó
  4. Update ref/heads/main lên commit mới

→ Kết quả: chỉ 1 commit = chỉ 1 lần Render deploy

KHÔNG dùng: GitHub Contents API (PUT từng file riêng lẻ)
  → Mỗi file = 1 commit = nhiều deploy = cancel nhau

Lệnh push mẫu (xem file data_nexorax_garden.txt nếu cần
tra cứu lại flow chi tiết).

CODE MẪU (JavaScript trong code_execution sandbox):
  ```javascript
  const RAILWAY_TOKEN = process.env["RAILWAY_TOKEN"];
  const PROJECT_ID = "2a399a3a-0878-4e4b-8c65-5a4bab0dc34a";

  async function railwayQuery(query) {
    const res = await fetch("https://backboard.railway.app/graphql/v2", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Authorization": `Bearer ${RAILWAY_TOKEN}`,
      },
      body: JSON.stringify({ query }),
    });
    return res.json();
  }

  // Lấy deployment mới nhất
  const deploys = await railwayQuery(`
    query { deployments(input: { projectId: "${PROJECT_ID}" }, first: 1) {
      edges { node { id status } }
    }}`);
  const deployId = deploys.data.deployments.edges[0].node.id;

  // Lấy logs
  const logs = await railwayQuery(`
    query { deploymentLogs(deploymentId: "${deployId}", limit: 200) {
      timestamp message severity
    }}`);
  console.log(logs.data.deploymentLogs
    .map(l => `[${l.severity}] ${l.message}`).join("\n"));
  ```
