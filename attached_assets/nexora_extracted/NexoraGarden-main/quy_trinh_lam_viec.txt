QUAN TRỌNG: Render tự động deploy mỗi khi có commit mới.
Nếu push nhiều commit liên tiếp → Render sẽ cancel hết, chỉ
giữ cái cuối cùng → gây lãng phí và log lộn xộn.

CÁCH PUSH ĐÚNG (dùng GitHub Git Tree API):
  1. Tạo blob cho từng file thay đổi
  2. Tạo 1 tree mới chứa tất cả file
  3. Tạo 1 commit duy nhất từ tree đó
  4. Update ref/heads/main lên commit mới

→ Kết quả: chỉ 1 commit = chỉ 1 lần Render deploy

KHÔNG dùng: GitHub Contents API (PUT từng file riêng lẻ)
  → Mỗi file = 1 commit = nhiều deploy = cancel nhau

Lệnh push mẫu (xem file data_nexorax_garden.txt nếu cần
tra cứu lại flow chi tiết).
