TOOL 1 — YOUTUBE / VIDEO DOWNLOADER
─────────────────────────────────────────────────────────────
URL: /tool/yt-downloader

Cách sử dụng:
  1. Dán link video (YouTube, TikTok, Instagram, Facebook, Twitter/X, v.v.)
  2. Nhấn tìm kiếm → hệ thống lấy thông tin video (tiêu đề, thumbnail, thời lượng, kênh)
  3. Chọn chất lượng: Tốt nhất / 1080p / 720p / 480p / 360p
  4. Nhấn "Tải xuống" → file tải về máy

Cách hoạt động kỹ thuật:
  Frontend → POST /api/yt/info?url=... → API Server
  API Server dùng yt-dlp để lấy metadata
  Khi tải: POST /api/yt/download với url + quality
  yt-dlp tải video, ffmpeg merge video+audio (nếu cần)
  Server stream file về trình duyệt

Chất lượng:
  - "Tốt nhất": tự động chọn chất lượng cao nhất có ffmpeg merge
  - 1080p: Full HD
  - 720p: HD
  - 480p: SD
  - 360p: Low quality, tải nhanh

Nền tảng hỗ trợ:
  YouTube, TikTok, Instagram, Facebook, Twitter/X, Streamable,
  Dailymotion, Vimeo, và 1000+ trang khác (yt-dlp hỗ trợ)

Giới hạn:
  - Không tải được video có DRM (Netflix, Disney+)
  - Một số video có tuổi người dùng giới hạn
  - Tốc độ phụ thuộc server Railway

─────────────────────────────────────────────────────────────
TOOL 2 — VIDEO TRIMMER (CẮT VIDEO)
─────────────────────────────────────────────────────────────
URL: /tool/video-trimmer

Cách sử dụng:
  1. Upload file video (kéo thả hoặc click chọn file, tối đa 500MB)
  2. Xem preview video trong trình duyệt
  3. Nhập thời gian bắt đầu: định dạng "1p30s" (1 phút 30 giây) hoặc "1:30"
  4. Nhập thời gian kết thúc (để trống = đến hết video)
  5. Nhấn "Cắt video" → tải về file đã cắt

Định dạng thời gian hỗ trợ:
  - "1p30s" hoặc "1p30" = 1 phút 30 giây
  - "1:30" = 1 phút 30 giây
  - "90" = 90 giây
  - "1:30:00" = 1 giờ 30 phút

Cách hoạt động kỹ thuật:
  Frontend upload file → POST /api/trim
  API Server nhận file (multer), chạy ffmpeg -ss [start] -to [end] -i input output
  ffmpeg cắt chính xác không re-encode (copy stream) → nhanh
  Server trả về file đã cắt để download

Giới hạn:
  - Tối đa 500MB mỗi file
  - Định dạng hỗ trợ: mp4, mkv, avi, mov, webm, v.v. (ffmpeg xử lý)
  - Thời gian xử lý phụ thuộc kích thước file

TOOL 3 — TEXT FORMATTER (AI TỰ ĐỘNG ĐỊNH DẠNG VĂN BẢN)
─────────────────────────────────────────────────────────────
URL: /tool/text-formatter
Trạng thái: LIVE — đang hoạt động đầy đủ

Mô tả:
  Công cụ AI tự động căn chỉnh và định dạng văn bản tiếng Việt chuẩn chuyên nghiệp.
  Hỗ trợ 3 chế độ: nhập text thủ công, upload file, và tạo nội dung bằng AI.
  Kết quả có thể copy hoặc xuất file .docx (Microsoft Word).

3 chế độ sử dụng:
  1. Upload File — kéo thả hoặc chọn file (.txt, .doc, .docx, .csv, .json, .html,
     .xml, .jpg, .png, .webp, .gif, .bmp) → AI đọc và định dạng lại nội dung
  2. Nhập text — gõ/dán văn bản thô (tối đa 2000 ký tự) → AI định dạng
  3. Tạo bằng AI — nhập yêu cầu, AI tạo nội dung mới theo đúng quy chuẩn

Quy tắc định dạng AI áp dụng:
  - Tiêu đề chính: căn giữa, chữ in hoa
  - Đề mục lớn: số La Mã (I. II. III.), in đậm
  - Đề mục nhỏ: số thường (1. 2. 3.), in đậm
  - Danh sách cấp 1: gạch ngang (-)
  - Danh sách cấp 2: thụt 4 khoảng trắng (+)
  - Đoạn văn thường: thụt đầu dòng 4 khoảng trắng
  - Câu hỏi trắc nghiệm: **Câu X:** ... A. B. C. D. ✔ Đáp án: X
  - Từ khóa quan trọng: in đậm **từ khóa**
  - Dòng kẻ phân cách: ---

Cách hoạt động kỹ thuật:
  Frontend → POST /api/format với body: { mode, content, mimeType, prompt }
  API Server nhận request → (nếu .docx: dùng mammoth extract text)
  Gọi GLM-4.6V-Flash qua api.llm7.io với system prompt định dạng
  Server stream SSE về frontend → hiển thị dần
  Kết quả render trực tiếp trên UI (preview ngay) + nút copy/xuất .docx

Xuất file .docx:
  Dùng thư viện docx (JavaScript) tạo file Word trực tiếp trên trình duyệt
  Hỗ trợ: căn giữa tiêu đề, in đậm đề mục, xuống dòng đúng chuẩn

Giới hạn:
  - Chế độ nhập text: tối đa 2000 ký tự
  - Chế độ upload file: xử lý file docx/txt/ảnh (ảnh qua AI đọc OCR)
  - Định dạng file hỗ trợ: .txt .doc .docx .csv .json .html .xml .jpg .png .webp .gif .bmp

PHẦN 8 — PHONG CÁCH LẬP TRÌNH VÀ DESIGN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Design System (Portfolio):
  - Nền: #050505 / #080808 (gần đen tuyền)
  - Font: Plus Jakarta Sans (weight 300 đến 800)
  - Màu sắc: Grayscale hoàn toàn (không có màu accent tím/xanh)
  - Border: rgba(255,255,255,0.08) đến rgba(255,255,255,0.18)
  - Background card: rgba(255,255,255,0.04) đến rgba(255,255,255,0.09)
  - Text chính: rgba(255,255,255,0.82)
  - Text phụ: rgba(255,255,255,0.35) đến rgba(255,255,255,0.55)
  - Border radius: 14px đến 22px (bo tròn nhiều)
  - Backdrop filter: blur(10px–24px) trên các panel

Kỹ thuật animation đặc trưng:
  1. Running Border (viền chạy): CSS mask-composite
     - .running-border::before dùng conic-gradient xoay
     - mask: content-box + normal, mask-composite: exclude
     - Chỉ thấy 1.5px viền, phần giữa trong suốt
     - Biến CSS: --rb-speed, --rb-color, --rb-radius

  2. Spin Rings (vòng xoay avatar): CSS @keyframes
     - ring-cw: xoay thuận chiều kim
     - ring-ccw: xoay ngược chiều kim
     - GPU composited (chỉ dùng transform: rotate)
     - Không tốn JS thread

  3. Scroll Reveal: Framer Motion whileInView + once: false
     - Elements ẩn khi vuốt lên, hiện khi vuốt xuống
     - Dùng margin: "0px 0px -100px 0px" để trigger sớm

  4. Spring Animation: Framer Motion type: "spring"
     - stiffness: 300–400, damping: 20–28
     - Tạo cảm giác vật lý tự nhiên

Nguyên tắc performance:
  - Tránh backdropFilter trong card animate nhiều
  - Chỉ animate khi element đang visible (useInView)
  - CSS animation thay Framer Motion cho các loop vô tận
  - Giảm số lượng running-border đồng thời (max 1–3)
